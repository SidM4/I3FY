Structure of Coding a Website

When a developer begins coding a website, they always have a wireframe with them. Now, what is a wireframe? They help a developer understand how the page is meant to look. A wireframe focuses on the user journey, to help them develop better understand the desired output. There are 3 main sections in a website page. A header, which is located right at the top of the screen. This contains navigation links to other pages and the site logo. Then comes the body. This is the main section of information. Everything goes here. All the information, images, headings, text, etc. At the very bottom of the page is the footer. The footer usually contains any copyright information or terms and conditions.



HTML Format: This is the basic structure to writing HTML code in any IDE. When we first open a page, we use the file extension ‘.html’. This lets the IDE know that when we open the file, it must be opened as an HTML file. Once the page is created, we must open a series of tags. To open any tag in HTML, it must be the tag name surrounded by a greater than and lesser than sign. Most tags require an opening and closing tag (as seen in the image below). This tag lets us give a name to our website for it to be displayed on the Tab Viewer at the top of any Web Browser. Along with the title tag, we should also open the head tag.
