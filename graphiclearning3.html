UI Concepts

When developing UI, we must keep in mind these fundamental concepts like Visual Design, White Space, Design System, Prototyping, Icons, Typography, Accessibility.

  • Visual Design: Over the many pages of the platform, there must be consistency in the colour scheme, sizing, typography, etc. These factors influence how easily information is interpreted on your platform.

  • White Space: This concept focuses on the amount of empty space on your site. Believe or not, having a certain amount of empty space can help improve concentration and focus on your platform.

  • Design System: A collection of all components or atoms that are reusable in the UI are called Design System. Buttons are an example of an element in a design file as the same format for the button is used across most pages.

  • Prototyping: This is a term widely used across many fields. Prototyping is an early testing solution that helps understand if there has to be any changes after some real-world usage. Many applications launch Beta versions of new releases. These Beta versions help them understand where their design is working and where it isn’t. They can then change the places it doesn’t work when it officially releases.

  • Icons: Little graphics on pages can help the user understand the information a lot better. Using icons can help improve efficiency in the platform as they are imminently recognisable by the user. They can also add some style and aesthetic to the site with a pop of colour. When developing a platform, you must use correct icons, as there can be a variety of types. The best way to find icons is by using icon packs. These packs have icons of the same format, like the same line texture, colour, pattern, weight, etc. This helps keep uniformity across the site.

  • Typography: Using appropriate fonts, font-sizing, font-colour is essential for user interactions. We start with the font-type. On the platform, there must be uniformity, but there can be variety as well. For example, on this site, I3FY, for all body text, we have used Roboto, but for headings and titles, we have used Montserrat. These 2 fonts are a great combination as they are alike. They aren’t very thick in terms of the font-weight, they have the same single line pattern, and they work with most colours. Next, the sizing. There must be variation in the sizing based on what you want the user to see first. Most deals and discount pop-ups have big, bold, bright text to catch the user’s attention. On this site, there is a variation in font size from 24px down to 12px. We have small text that doesn’t require much attention, then we have large titles that help the user direct themselves to their interested topic. This is good typography.

  • Accessibility: All pages, all elements and all information must be accessible. We have spoken about the practicality of using different elements to gain the users attention but what happens after we gain their attention? All buttons, redirects and clickable features must be accessible. It shouldn’t be too big, it shouldn’t be too small. It should work on all types of screens, from large monitors to small phones. Having good user accessibility is essential to every platform, ever.
